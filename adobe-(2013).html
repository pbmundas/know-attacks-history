Attack Timeline

Date/Time Action Actor/Details Evidence/Impact
Mid-August 2013 Initial breach: Attackers exploit vulnerabilities in Adobe’s network, likely outdated ColdFusion servers. Cybercriminals (suspected, financially motivated) Gained access to customer data and source code repositories.
August-September 2013 Data collection and exfiltration: Stolen 2.9 million (later 38 million) user records and source code. Attackers Included names, encrypted passwords, credit card details, emails, and source code for Acrobat, ColdFusion, Photoshop.
September 17, 2013 Adobe detects breach after server capacity alert reveals decryption attempts. Adobe Unauthorized access to payment card data detected; investigation begins.
Late September 2013 Source code discovered on hacker’s server by KrebsOnSecurity and Hold Security. Brian Krebs, Alex Holden 40GB of Adobe source code found, indicating broader compromise.
October 3, 2013 Adobe publicly announces breach, confirming 2.9 million accounts affected. Adobe Password resets initiated; banks and customers notified.
October 29, 2013 Breach scope expanded: 38 million active accounts confirmed compromised. Adobe Additional inactive/test accounts affected; total ~153 million records.
November 2013 Fallout: Stolen data used for phishing and password reuse attacks; source code risks persist. Cybercriminals Impacted users of other platforms (e.g., Facebook, Diapers.com) due to password reuse.
November 7, 2016 Legal resolution: Adobe settles with 15 state attorneys general for $1 million. Adobe Implemented new security policies; reputational damage persists.

Dwell Time Analysis

Dwell time was approximately 30-45 days (mid-August to September 17, 2013), the period attackers operated undetected in Adobe’s network, allowing data and source code exfiltration.

Step 1: Initial Access (Mid-August) Step 2: Collection (August-September) Step 3: Exfiltration (August-September) Step 4: Detection (September 17)
Exploited vulnerabilities, likely in outdated ColdFusion servers.
What went wrong: Unpatched software, weak network segmentation.		 Queried databases for user data; accessed source code repositories.
Factor: No anomaly detection for mass data access.		 Exfiltrated 38 million records and 40GB source code.
Factor: No egress filtering or DLP for large data transfers.		 Server capacity alert flagged decryption attempts.
Total time: Enabled massive data and IP theft.

Dwell Time Insight: The dwell time was like a silent raid on a vault, unnoticed due to inadequate monitoring. Experts: Median dwell time in 2013 was ~150 days (per Mandiant); Adobe’s 30-45 days was relatively short but impactful due to poor encryption and source code loss.

Learnings

Root Cause Analysis Impact Assessment Detection Opportunities Response and Remediation Attribution Analysis Lessons Learned
Using 5 Whys: Why breached? Vulnerable servers. Why vulnerable? Outdated software (e.g., ColdFusion). Why outdated? No patch management. Why no patching? Siloed IT/security teams. Why siloed? Prioritized product over security.
Other causes: Weak encryption (no salting), no DLP, inadequate network monitoring.
Beginners: Root causes trace security gaps.
Experts: Fishbone factors: Technology (outdated software), Process (no DLP), People (security culture).		 Data: 38 million active accounts, ~153 million total, 40GB source code.
Financial: $1M settlement, millions in remediation.
Reputational: Trust erosion; impacted Adobe’s cloud transition.
Follow-on: Password reuse attacks, potential zero-day exploits.
Beginners: Impacts hit users and brand.
Experts: BIA metrics: RPO violated due to source code loss.		 Missed: Large data queries could have been caught with SIEM; exfiltration with DLP; server issues with monitoring.
Opportunities: Deploy SIEM for query spikes, DLP for egress, patch management for servers.
Failures: No encryption audits, short log retention.
Beginners: Detection is like a watchdog.
Experts: Sigma rule example: detection: logsource: category: network | selection: DataVolume > 1GB AND Destination: external.		 Immediate: Password resets, credit monitoring, notified banks.
Long-term: Adopted salted hashing, integrated security teams, implemented DLP.
Beginners: Response is damage control.
Experts: Followed NIST IR: Identification (September), Containment (October), Eradication (post-2013).		 Suspected: Financially motivated cybercriminals.
TTPs: Server exploitation, data/source code theft align with cybercrime groups.
Beginners: Attribution is identifying culprits.
Experts: TTPs suggest Eastern European or Russian actors; limited IOCs due to encrypted data.		 Patch servers promptly.
